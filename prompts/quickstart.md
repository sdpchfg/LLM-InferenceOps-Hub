# LLM InferenceOps Hub 快速入门

## 适用对象

| 级别 | 描述 | 使用建议 |
|------|------|----------|
| **小白** | 完全不了解LLM部署 | 从"环境准备"开始，按顺序阅读 |
| **有基础** | 了解基本概念 | 直接查阅"问题诊断"模板 |
| **高级用户** | 遇到复杂问题 | 使用"故障排查"和"性能优化"章节 |

## 快速开始

### 第一步：识别你的级别

你是哪类用户？
- 小白：需要从零开始学习
- 有基础：需要解决特定问题
- 高级用户：需要深度故障排查

### 第二步：使用问诊模板

无论你是哪类用户，都可以使用以下模板提问：

```
问题：[简要描述]
环境：操作系统 + GPU型号
已尝试：[你尝试过的]
```

### 第三步：获取解决方案

系统会根据你的问题类型和级别，提供：
- **小白**：详细的分步指导
- **有基础**：核心解决方案 + 可选深入内容
- **高级用户**：直接可用的命令和配置

## 推荐学习路径

### 小白路线

1. **环境准备** → 安装驱动、Docker、CUDA
2. **框架选择** → 选择vLLM（推荐入门）
3. **第一个模型** → 部署一个简单模型
4. **API使用** → 学习调用API

### 有基础路线

1. **性能调优** → 优化推理速度
2. **多卡部署** → 扩展到多GPU
3. **生产部署** → 添加监控和负载均衡

### 高级路线

1. **定制量化** → 自定义量化方案
2. **内核优化** → CUDA内核调优
3. **集群部署** → Kubernetes大规模部署

## 常见问题速查

| 问题 | 快速答案 |
|------|----------|
| "哪个框架最好？" | vLLM适合大多数场景 |
| "需要多少显存？" | 7B模型至少16GB，70B至少80GB×4 |
| "如何加速推理？" | 启用FlashAttention + 量化 |
| "服务启动失败？" | 检查nvidia-smi和docker logs |
| "如何降低延迟？" | 启用Speculative Decoding |

## 框架推荐

| 场景 | 推荐框架 | 原因 |
|------|----------|------|
| 入门首选 | vLLM | 简单易用，性能优秀 |
| NVIDIA优化 | TensorRT-LLM | 最强性能，需要专业调优 |
| 本地运行 | Ollama | 开箱即用，跨平台 |
| CPU推理 | llama.cpp | 无GPU要求 |
| 生产环境 | TGI | 稳定可靠，功能丰富 |
